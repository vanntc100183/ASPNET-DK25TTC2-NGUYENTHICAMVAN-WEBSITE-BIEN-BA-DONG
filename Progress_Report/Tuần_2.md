# BÁO CÁO TIẾN ĐỘ ĐỒ ÁN - TUẦN 2
**Đề tài:** Xây dựng website Quảng bá du lịch biển Ba Động
**Thời gian thực hiện:** 29/06/2026 - 05/07/2026
**Giai đoạn:** Phân tích thiết kế hệ thống và Cơ sở dữ liệu

## 1. Nội dung công việc
- Phân tích yêu cầu nghiệp vụ của hệ thống: Quản lý tin tức du lịch, quản lý danh mục địa điểm, dịch vụ lưu trú, ẩm thực tại biển Ba Động và module phản hồi từ du khách.
- Xác định các tác nhân (Actor) tham gia vào hệ thống và xây dựng biểu đồ Use Case tổng thể, Use Case chi tiết cho từng phân hệ.
- Thiết kế biểu đồ lớp (Class Diagram) và biểu đồ tuần tự (Sequence Diagram) cho các chức năng cốt lõi.
- Thiết kế mô hình quan hệ thực thể (ERD - Entity Relationship Diagram) cho cơ sở dữ liệu.
- Xây dựng cấu trúc các bảng trong cơ sở dữ liệu SQL Server: Khóa chính, khóa ngoại, các ràng buộc toàn vẹn dữ liệu.

## 2. Tài liệu liên quan
- Giáo trình Phân tích và Thiết kế hệ thống thông tin.
- Tài liệu hướng dẫn thiết kế cơ sở dữ liệu quan hệ chuẩn hóa (1NF, 2NF, 3NF).
- Các công cụ hỗ trợ vẽ sơ đồ như Microsoft Visio hoặc draw.io.

## 3. Khó khăn gặp phải
- Việc phân tách các thực thể và xác định mối quan hệ (1-n, n-n) giữa các bảng quản lý tour du lịch, dịch vụ đi kèm và tài khoản người dùng còn một số điểm chưa tối ưu, dễ dẫn đến trùng lặp dữ liệu.
- Chuẩn hóa cơ sở dữ liệu lên mức 3NF đòi hỏi phải tư duy chặt chẽ để đảm bảo hiệu năng truy vấn sau này.

## 4. Kết quả đạt được
- Hoàn thiện tài liệu phân tích thiết kế hệ thống bao gồm đầy đủ các biểu đồ Use Case, ERD.
- Khởi tạo thành công cơ sở dữ liệu vật lý trên SQL Server với hệ thống bảng dữ liệu chuẩn hóa, sẵn sàng kết nối với source code.
